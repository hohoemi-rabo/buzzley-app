## 受け入れ条件
- エラー発生時にアプリがクラッシュしない
- ユーザーに分かりやすいエラーメッセージが表示される
- リトライ可能なエラーは自動リトライされる
- エラー情報が適切にログされる

## 技術メモ
- Error Boundary: React 18のError Boundary使用
- トースト通知: react-hot-toastなどを検討
- エラーログ: console.errorとSentryの将来的導入
